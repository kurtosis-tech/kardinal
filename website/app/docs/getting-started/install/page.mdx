# Installation

In this guide, you'll:
1. Install the Kardinal CLI on your laptop
2. Set up a development Kubernetes cluster
3. Install the Kardinal Manager on your cluster, using the CLI

### Step 1: Install Kardinal

To install Kardinal, run the following command:

```bash
curl get.kardinal.dev -sL | sh
```

### Step 2: Set up a development Kubernetes cluster

All you need is a Kubernetes cluster with Istio enabled, and `kubectl` installed on your machine, pointing to your cluster.

For instructions on how to set up a local cluster that meets these requirements, see below:

<Collapsible title="Minikube with Docker">

Install the following on your machine:

- [Docker](https://docs.docker.com/get-docker/)
- [Minikube](https://minikube.sigs.k8s.io/docs/start/)
- [Kubectl](https://kubernetes.io/docs/tasks/tools/install-kubectl/)
- [Istio](https://istio.io/latest/docs/setup/getting-started/)



Then, run Minikube with Istio enabled:

```bash
minikube start --driver=docker --cpus=10 --memory 8192 --disk-size 32g
minikube addons enable ingress
minikube addons enable metrics-server
istioctl install --set profile=default -y
```

</Collapsible>

### Step 3: Deploy the Kardinal Manager to your cluster

Make sure that `kubectl` is pointing to your cluster, and then run the following command:

```bash
kardinal manager deploy kloud-kontrol
```

### Step 4: Deploy your application

To deploy a sample application and see how Kardinal works, run:

```bash
curl -O https://raw.githubusercontent.com/kurtosis-tech/kardinal-playground/main/obd-demo.yaml
kardinal deploy obd-demo.yaml
```

To see the sample application architecture in the Kardinal dashboard, run:
```bash
kardinal dashboard
```

To access the frontend of your application, run:
```bash
minikube tunnel
```
in a new terminal window, and then open https://prod.app.localhost in your browser.

To deploy your application and create your first flow, follow our guide at [create your first flow](/docs/getting-started/own-app).
